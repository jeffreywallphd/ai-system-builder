# AI Companion: Contributors Documentation Router

## Audience
- AI assistants routing implementation questions to contributor guidance.
- Engineers checking where contributor workflows are documented.

## Purpose
- Entry point for contributor-facing extension and implementation workflow docs.

## Belongs Here
- Contributor guides for feature extensions and safe change workflows.
- Guardrails/checklists that constrain implementation patterns.
- Workflow references that explain how to apply architecture contracts in code changes.

## Does Not Belong Here
- Operations runbooks or runtime administration procedures.
- ADR records and design-decision history.
- Historical baseline artifacts.
