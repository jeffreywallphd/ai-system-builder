# AI Companion: Context Governance Directory

## Audience
- AI assistants and maintainers validating context-asset quality.
- Engineers proposing context pack or routing updates.

## Purpose
- Centralize governance standards that keep context assets stable and low-noise.

## Belongs Here
- Governance policy for context lifecycle and review cadence.
- Ownership and update expectations for packs and routing maps.
- Change-management guidance for contract artifacts.

## Does Not Belong Here
- Subsystem architecture contracts.
- Contributor coding workflows.
- Operational incident runbooks.
