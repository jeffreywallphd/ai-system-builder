# Context Governance Directory

## Audience
- Maintainers reviewing context quality and lifecycle hygiene.
- Engineers and assistants proposing changes to pack or routing artifacts.

## Purpose
- Define governance expectations so context artifacts remain accurate, low-noise, and maintainable.

## Belongs Here
- Context governance standards and review policies.
- Ownership and maintenance cadence guidance for context artifacts.
- Change control guidance for pack and routing contracts.

## Does Not Belong Here
- Detailed architecture subsystem contracts.
- Feature implementation tutorials.
- Runtime operations or incident procedures.
