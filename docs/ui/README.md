# UI Documentation Router

## Audience
- Frontend engineers and reviewers working on studio UX behavior.

## Purpose
- UI behavior contracts and UX implementation notes.

## Belongs Here
- UI state/flow behavior documentation.
- Frontend implementation contracts and UX conventions.
- UI-specific notes that guide renderer-side changes.

## Does Not Belong Here
- Non-UI architecture baselines.
- Runtime operations runbooks.
- General contributor workflow checklists.
